title: "Quickstart"
description: "Make your first type-safe HTTP request with okfetch in under five minutes."
---

This guide walks you through installing `@okfetch/fetch`, making a validated request, and handling the `Result`.

<Steps>
  <Step title="Install the package">
    Install `@okfetch/fetch` along with its two peer dependencies: `better-result` for the `Result` type, and `zod` for runtime validation.

    <CodeGroup>
      ```bash npm
      npm install @okfetch/fetch better-result zod
      ```

      ```bash yarn
      yarn add @okfetch/fetch better-result zod
      ```

      ```bash pnpm
      pnpm add @okfetch/fetch better-result zod
      ```

      ```bash bun
      bun add @okfetch/fetch better-result zod
      ```
    </CodeGroup>
  </Step>

  <Step title="Make your first request">
    Import `okfetch` and define a Zod schema for the expected response shape. Pass the schema as `outputSchema` — okfetch will validate the response before it reaches your code.

    ```typescript
    import { okfetch } from "@okfetch/fetch";
    import { z } from "zod/v4";

    const todoSchema = z.object({
      completed: z.boolean(),
      id: z.number(),
      title: z.string(),
      userId: z.number(),
    });

    const result = await okfetch("https://jsonplaceholder.typicode.com/todos/1", {
      outputSchema: todoSchema,
    });
    ```

    `okfetch` always returns a `Result<T, E>` — it never throws for HTTP failures, timeouts, or validation errors.
  </Step>

  <Step title="Handle the Result">
    Use `.match()` to handle success and failure in a single expression:

    ```typescript
    result.match({
      ok: (todo) => console.log(todo.title),       // todo is fully typed
      err: (error) => console.error(error._tag, error.message),
    });
    ```

    Or use the guard methods if you prefer early-return style:

    ```typescript
    if (result.isErr()) {
      console.error(result.error._tag, result.error.message);
      return;
    }

    const todo = result.value; // typed as { completed: boolean; id: number; title: string; userId: number }
    console.log(todo.title);
    ```

    <Tip>
      Every error has a `_tag` string you can switch on to narrow the type: `"FetchError"`, `"TimeoutError"`, `"ApiError"`, `"ParseError"`, `"ValidationError"`, `"PluginError"`. See [Error classes](/reference/fetch/errors) for details.
    </Tip>
  </Step>

  <Step title="Add a typed API client (optional)">
    For applications with many repeated API calls, use `@okfetch/api` to define your endpoints once and generate a fully typed client.

    <CodeGroup>
      ```bash npm
      npm install @okfetch/api @okfetch/fetch better-result zod
      ```

      ```bash bun
      bun add @okfetch/api @okfetch/fetch better-result zod
      ```
    </CodeGroup>

    ```typescript
    import { createApi, createEndpoints } from "@okfetch/api";
    import { z } from "zod/v4";

    const todoSchema = z.object({
      completed: z.boolean(),
      id: z.number(),
      title: z.string(),
      userId: z.number(),
    });

    const endpoints = createEndpoints({
      todos: {
        get: {
          method: "GET",
          output: todoSchema,
          params: z.object({ id: z.number() }),
          path: "/todos/:id",
        },
        create: {
          body: z.object({ title: z.string().min(1), userId: z.number() }),
          method: "POST",
          output: todoSchema,
          path: "/todos",
        },
      },
    });

    const api = createApi({
      baseURL: "https://jsonplaceholder.typicode.com",
      endpoints,
    });

    // Fully typed — params, body, and response are all inferred from schemas
    const result = await api.todos.get({ params: { id: 1 } });
    ```

    <Note>
      The API client returns the same `Result<T, E>` as `okfetch()` — the same error handling patterns apply.
    </Note>
  </Step>
</Steps>
